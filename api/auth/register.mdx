## Endpoint

```
POST /auth/register
```

Creates a new user account in the system. The handle is automatically converted to a URL-friendly slug format.

## Request Body

<ParamField body="handle" type="string" required>
  User's unique handle. Will be converted to a slug format (lowercase, hyphens instead of spaces).
  
  **Validation:** Cannot be empty
</ParamField>

<ParamField body="username" type="string" required>
  Display name for the user.
  
  **Validation:** Cannot be empty
</ParamField>

<ParamField body="email" type="string" required>
  User's email address.
  
  **Validation:** Must be a valid email format
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Will be hashed before storage.
  
  **Validation:** Must be at least 6 characters long
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

## Status Codes

- `201` - User created successfully
- `400` - Validation error (invalid input)
- `409` - Conflict (email or handle already exists)

## Error Responses

### Email Already Exists
```json
{
  "message": "Email already exists"
}
```
**Status Code:** `409`

### Handle Already Exists
```json
{
  "message": "Handle already exists"
}
```
**Status Code:** `409`

### Validation Errors

<Warning>
Validation errors return a `400` status code with specific error messages:
- "Handle is required"
- "Username is required"
- "Invalid email format"
- "Password must be at least 6 characters long"
</Warning>

## Example Request

```bash curl
curl -X POST http://localhost:4000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "handle": "john-dev",
    "username": "John Developer",
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

## Example Response

### Success Response (201)
```json
{
  "message": "User created successfully"
}
```

<Note>
The handle field is automatically converted to a slug format. For example, "John Dev" becomes "john-dev".
</Note>

<Note>
Passwords are securely hashed before being stored in the database.
</Note>
