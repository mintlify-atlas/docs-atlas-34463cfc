This guide will help you set up and run the Dev-Tree Backend API on your local machine.

## Prerequisites

Before you begin, ensure you have the following installed:

<Steps>
  <Step title="Node.js">
    Install Node.js (version 16 or higher recommended)
    ```bash
    node --version
    ```
  </Step>
  
  <Step title="MongoDB">
    You'll need a MongoDB database. You can use:
    - [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) (cloud solution)
    - Local MongoDB installation
  </Step>
  
  <Step title="Cloudinary Account">
    Create a free [Cloudinary account](https://cloudinary.com/) for image uploads
  </Step>
</Steps>

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/HeymerDev/dev-tree-backend.git
    cd dev-tree-backend
    ```
  </Step>
  
  <Step title="Install dependencies">
    Install all required packages using npm:
    ```bash
    npm install
    ```
    
    This will install:
    - **express** - Web framework
    - **mongoose** - MongoDB ODM
    - **jsonwebtoken** - JWT authentication
    - **bcrypt** - Password hashing
    - **cloudinary** - Image storage
    - **express-validator** - Request validation
    - And more...
  </Step>
</Steps>

## Environment Configuration

Create a `.env` file in the root directory with the following variables:

<CodeGroup>
```bash .env
# Server Configuration
PORT=3000

# Database
MONGO_URI=mongodb://localhost:27017/devtree
# Or for MongoDB Atlas:
# MONGO_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/devtree

# Frontend URL (for CORS)
FRONTEND_URL=http://localhost:5173

# JWT Secret
JWT_SECRET=your-super-secret-jwt-key-change-this

# Cloudinary Configuration
CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
```
</CodeGroup>

<Note>
  The environment variables are loaded automatically via the `dotenv` package in `src/utils/loadEnv.ts:1`
</Note>

<Warning>
  Never commit your `.env` file to version control. Keep your secrets safe!
</Warning>

### Getting Your Cloudinary Credentials

1. Sign up at [cloudinary.com](https://cloudinary.com/)
2. Go to your Dashboard
3. Copy the **Cloud Name**, **API Key**, and **API Secret**
4. Paste them into your `.env` file

## Database Setup

The application connects to MongoDB automatically when the server starts. The connection logic is in `src/config/db.ts:3`:

```typescript
export const connectDB = async () => {
    try {
        const conection = await mongoose.connect(process.env.MONGO_URI as string);
        const url = `${conection.connection.host}:${conection.connection.port}`;
        console.log("MongoDB connected successfully");
    } catch (error) {
        console.error("MongoDB connection failed", error);
        process.exit(1);
    }
}
```

<Info>
  The database connection is established automatically on server startup. No manual migration steps required!
</Info>

## Running the Server

Dev-Tree Backend provides multiple ways to run the server:

<CodeGroup>
```bash Development
npm run dev
```

```bash Development with API access
npm run dev:api
```

```bash Production
npm run build
npm start
```
</CodeGroup>

### Development Mode

For local development with auto-reload:

```bash
npm run dev
```

This command:
- Runs the server with `nodemon` for automatic restarts
- Watches for file changes in the `src/` directory
- Serves the API on `http://localhost:3000`

<Info>
  The `--api` flag in `npm run dev:api` allows API testing tools (Postman, Insomnia) to bypass CORS restrictions. See `src/config/cors.ts:7`
</Info>

### Production Mode

For production deployment:

<Steps>
  <Step title="Build the project">
    Compile TypeScript to JavaScript:
    ```bash
    npm run build
    ```
    This generates compiled code in the `dist/` directory
  </Step>
  
  <Step title="Start the server">
    Run the production build:
    ```bash
    npm start
    ```
  </Step>
</Steps>

## Testing the API

Once the server is running, you can test the API endpoints:

### Health Check

Verify the server is running:

```bash
curl http://localhost:3000/
```

### Register a User

Create a new user account:

```bash
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "handle": "johndoe",
    "username": "John Doe",
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

### Login

Authenticate and receive a JWT token:

```bash
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

<Note>
  Save the JWT token from the login response. You'll need it for authenticated endpoints.
</Note>

### Get User Profile

Fetch authenticated user data:

```bash
curl http://localhost:3000/user \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### Get Public Profile

View a user's public profile by their handle:

```bash
curl http://localhost:3000/johndoe
```

## API Endpoints Overview

Here's a quick reference of available endpoints (defined in `src/router.ts`):

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| POST | `/auth/register` | Register a new user | No |
| POST | `/auth/login` | Login and get JWT token | No |
| GET | `/user` | Get authenticated user profile | Yes |
| GET | `/:handle` | Get user profile by handle | No |
| POST | `/search` | Search user by handle | No |
| PATCH | `/user` | Update user profile | Yes |
| POST | `/user/image` | Upload profile image | Yes |

## Troubleshooting

<AccordionGroup>
  <Accordion title="MongoDB connection failed">
    **Error:** `MongoDB connection failed`
    
    **Solutions:**
    - Verify your `MONGO_URI` is correct in `.env`
    - Ensure MongoDB is running (if using local installation)
    - Check network connectivity to MongoDB Atlas
    - Verify your MongoDB Atlas IP whitelist includes your IP
  </Accordion>
  
  <Accordion title="CORS errors">
    **Error:** `Error de CORS`
    
    **Solutions:**
    - Verify `FRONTEND_URL` in `.env` matches your frontend origin
    - Use `npm run dev:api` for API testing tools
    - Check CORS configuration in `src/config/cors.ts`
  </Accordion>
  
  <Accordion title="Port already in use">
    **Error:** `Port 3000 already in use`
    
    **Solutions:**
    - Change the `PORT` in your `.env` file
    - Kill the process using port 3000:
      ```bash
      # Find the process
      lsof -i :3000
      # Kill it
      kill -9 <PID>
      ```
  </Accordion>
  
  <Accordion title="Cloudinary upload fails">
    **Error:** Image upload returns an error
    
    **Solutions:**
    - Verify all Cloudinary credentials in `.env`
    - Check the configuration in `src/config/cloudinary.ts:3`
    - Ensure your Cloudinary account is active
  </Accordion>
</AccordionGroup>

## Development Tips

<Tip>
  Use `npm run dev:api` during development to allow requests from API testing tools like Postman or Insomnia without CORS restrictions.
</Tip>

<Tip>
  The server uses TypeScript with `nodemon` for hot reloading. Save any file in the `src/` directory to automatically restart the server.
</Tip>

<Tip>
  JWT tokens are used for authentication. Include the token in the `Authorization` header as `Bearer <token>` for protected routes.
</Tip>
